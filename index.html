Guía Completa del Procesador 8086/88

Este documento proporciona una visión general completa del procesador Intel 8086/88, incluyendo su arquitectura, registros, modos de direccionamiento e instrucciones. Ideal para estudiantes y profesionales interesados en la arquitectura de computadoras y programación en lenguaje ensamblador.

Bus de datos: 16 bits (8086) / 8 bits (8088)
Bus de direcciones: 20 bits (1 MB de memoria)
Año de lanzamiento: 1978
Fabricante: Intel Corporation

El lenguaje ensamblador es un conjunto de palabras que indican al ordenador lo que tiene que hacer. A diferencia de otros lenguajes, cada instrucción en ensamblador tiene una correspondencia exacta con una operación en el procesador.

Tamaño de los datos

  • 4 bits: nibble
  • 8 bits: byte
  • 16 bits: word
  • 32 bits: dword

Almacenamiento de datos

El 8086/88 usa el formato "little endian": el byte menos significativo (LSB) del dato es guardado en la parte baja de la memoria.

Segmentación

El 8086/88 tiene un bus de datos de 16 bits y uno de direcciones de 20 bits (accede a 1 MB). Para aprovechar la capacidad de direccionamiento, cada dirección se especifica como un segmento y un desplazamiento.

Dirección Efectiva = Segmento (desplazado 4 bits) + Desplazamiento

Registros Internos

El 8086/88 dispone de 4 registros de datos, 4 registros de segmento, 5 registros de índice y 1 registro de estado.

Los registros de datos son de 16 bits, aunque están divididos, lo que permite su acceso en 8 bits.

Registros de Segmento: Los Mapas de la Memoria

Los registros de segmento son de 16 bits y actúan como "mapas" que permiten al procesador 8086/88 navegar por su espacio de memoria de 1MB utilizando direcciones de 20 bits, a pesar de que el bus de direcciones interno es de 16 bits.

Cada registro de segmento apunta al inicio de un área de memoria (segmento) de 64 KB. Para formar una dirección física completa, el procesador:

  1. Toma el valor del registro de segmento
  2. Lo desplaza 4 bits a la izquierda (multiplica por 16)
  3. Suma un desplazamiento (offset) para obtener la dirección efectiva

Contiene la dirección base del segmento de código.

Trabaja con IP para ubicar instrucciones.

CS: Code Segment (El mapa del código)

"CS es el Code Segment. Este es el mapa que nos lleva al barrio donde está el código del programa, o sea, las instrucciones que el procesador tiene que ejecutar. CS trabaja en equipo con el registro IP —el Instruction Pointer—, que nos dice qué 'casa' dentro de ese barrio visitar para leer la próxima instrucción."

Ejemplo de cálculo de dirección

Si CS es 0x1000 y IP es 0x0020, el procesador hace un cálculo:

0x1000 × 16 + 0x0020 = 0x10020

¡Esa es la dirección exacta donde está la instrucción!

CS cambia cuando hacemos saltos lejanos (FAR JMP o FAR CALL), como si viajáramos a otro barrio del código.

Ejemplos de uso

JMP FAR PTR OTRA_RUTINA   ; Salta a otro segmento, actualiza CS e IP
                CALL FAR PTR PROCEDIMIENTO  ; Llama a procedimiento en otro segmento

DS: Data Segment (El mapa de los datos)

"DS es el Data Segment. Este nos guía al barrio donde están los datos con los que trabajamos, como variables o números. Si quiero leer un valor en memoria, uso DS con un desplazamiento."

Ejemplo de cálculo de dirección

Si DS es 0x2000 y el desplazamiento es 0x0100, la dirección es:

0x2000 × 16 + 0x0100 = 0x20100

Ahí está el dato. En ensamblador, lo escribimos como MOV AX, [DS:0x0100].

DS es el mapa que usamos por defecto para casi todas las operaciones con datos.

Ejemplos de uso

MOV AX, [SI]          ; Implícitamente usa DS como segmento
                MOV BX, variable     ; DS:offset de "variable"
                MOV CX, [BX+10]      ; DS:(BX+10) es la dirección efectiva

SS: Stack Segment (El mapa de la pila)

"SS es el Stack Segment. Este es el mapa del barrio de la pila, un lugar especial donde guardamos cosas temporalmente, como cuando apilamos platos en la cocina. SS trabaja con el registro SP —el Stack Pointer—, que señala la última 'casa' ocupada en la pila."

Ejemplo de cálculo de dirección

Si SS es 0x3000 y SP es 0xFFFE, la dirección es:

0x3000 × 16 + 0xFFFE = 0x3FFFE

Ahí está el último dato apilado.

Cuando usamos PUSH o POP, el procesador ajusta SP para apilar o desapilar cosas en este barrio.

Ejemplos de uso

PUSH AX           ; Guarda AX en SS:SP, luego decrementa SP
                POP BX            ; Incrementa SP, luego recupera valor en BX
                MOV [BP+4], CX    ; BP usa SS por defecto, accede a parámetros

ES: Extra Segment (El mapa extra)

"ES es el Extra Segment. Este es como un mapa de repuesto, un barrio adicional para datos que no caben en DS. Lo usamos en operaciones especiales, como mover cadenas de caracteres."

Ejemplo de cálculo de dirección

Si ES es 0x4000 y DI —el Destination Index— es 0x0050, la dirección es:

0x4000 × 16 + 0x0050 = 0x40050

ES es súper útil cuando necesitamos trabajar con dos barrios de datos al mismo tiempo.

Ejemplos de uso

MOVS BYTE PTR ES:[DI], DS:[SI]  ; Copia de DS:SI a ES:DI
                LES BX, DWORD PTR [SI]          ; Carga puntero en ES:BX
                MOV ES:[DI], AX                 ; Almacena AX en ES:DI

La Ciudad de la Memoria

Para entender mejor los segmentos, piensa en la memoria como una ciudad gigante. Cada registro de segmento es como un mapa de diferente color que te guía por un "barrio" específico de 64KB.

Casos de Uso Prácticos

Carga de programa

Al iniciar un programa, el sistema operativo carga el archivo ejecutable y configura CS:IP para apuntar al punto de entrada del programa.

; Inicialización típica de programa
                MOV AX, @CODE   ; Carga segmento de código en AX
                MOV CS, AX      ; Configura CS (solo permitido en ciertas circunstancias)
                JMP INICIO      ; Establece IP

Acceso a variables y arrays

DS se usa para acceder a datos estáticos, variables globales y arrays definidos en el segmento de datos.

; Configuración de DS al inicio del programa
                MOV AX, @DATA   ; Carga segmento de datos en AX
                MOV DS, AX      ; Configura DS
                
                ; Uso de un array
                MOV SI, OFFSET array   ; SI = desplazamiento del array
                MOV AL, [SI]           ; Carga primer elemento (DS:SI)

Llamadas a procedimientos

Cuando se llama a un procedimiento, la dirección de retorno se guarda en la pila (SS:SP).

; Llamada a procedimiento
                CALL Procedimiento   ; PUSH IP (dirección de retorno) a SS:SP
                
                ; Acceso a parámetros mediante BP
                Procedimiento PROC
                  PUSH BP          ; Guarda BP
                  MOV BP, SP       ; BP = SP (apunta a pila)
                  MOV AX, [BP+4]   ; Accede a parámetro en pila
                  POP BP           ; Restaura BP
                  RET              ; Retorno (POP IP)
                Procedimiento ENDP

Operaciones con cadenas

ES se usa principalmente para operaciones de copia entre segmentos diferentes, especialmente con operaciones de cadenas.

; Configuración para copia de cadena
                MOV AX, BufferDestino   ; Segmento destino
                MOV ES, AX              ; Configura ES
                MOV DI, 0               ; Desplazamiento destino
                
                MOV AX, BufferOrigen    ; Segmento origen 
                MOV DS, AX              ; Configura DS
                MOV SI, 0               ; Desplazamiento origen
                
                MOV CX, 10              ; Contador (10 bytes)
                REP MOVSB               ; Copia DS:SI -> ES:DI

Preguntas Frecuentes

+

¿Por qué el 8086/88 usa segmentación en lugar de direccionamiento lineal?

La segmentación fue una solución inteligente para extender el espacio de direcciones del 8086/88 más allá de los 64KB que permitirían sus registros de 16 bits. Esto permitió acceder a 1MB de memoria mientras se mantenía compatible con la arquitectura de 16 bits, sin necesidad de registros más grandes.

Además, la segmentación facilita la organización lógica del programa al separar código, datos y pila en áreas distintas, lo que mejora la modularidad y la gestión de memoria.

+

¿Qué ocurre si una dirección efectiva supera 1MB?

El 8086/88 solo puede direccionar 1MB (2^20 bytes) en total. Si el cálculo de segmento × 16 + desplazamiento excede 0xFFFFF (1,048,575), ocurre un "wraparound" o desbordamiento. Esto significa que la dirección vuelve al inicio del espacio de memoria (0) y continúa desde allí.

Por ejemplo, si la dirección calculada es 0x10FFFF (1,114,111), el valor efectivo sería 0x0FFFF (65,535), ya que solo se utilizan los 20 bits menos significativos.

+

¿Puedo modificar directamente los registros de segmento?

Los registros de segmento no pueden ser destino directo de la mayoría de instrucciones. Para modificarlos:

  1. Se debe cargar primero el valor en un registro general (como AX)
  2. Luego transferirlo al registro de segmento con MOV

Por ejemplo: MOV AX, valor seguido de MOV DS, AX

CS es especial y generalmente solo cambia mediante JMP FAR o CALL FAR, o al cargar un valor con instrucciones como RETF.

+

¿Qué es un override de segmento y cuándo usarlo?

Un override de segmento permite especificar explícitamente qué registro de segmento usar para una operación de memoria, anulando el que se usaría por defecto.

Por defecto:

  • CS se usa implícitamente para acceder a instrucciones con IP
  • DS se usa para la mayoría de accesos a datos con SI, BX, DI
  • SS se usa para SP y operaciones de pila, y para accesos usando BP

Ejemplo de override: MOV AX, ES:[BX] - Usa ES en lugar de DS

Estos registros son usados como índices por algunas instrucciones. También pueden ser usados como operandos (excepto IP).

IP (índice de programa)

Almacena el desplazamiento dentro del segmento de código. Junto al registro CS apunta a la próxima instrucción.

SI (índice de origen)

Almacena el desplazamiento del operando de origen en memoria.

DI (índice de destino)

Almacena el desplazamiento del operando de destino en memoria.

SP (índice de pila)

Almacena el desplazamiento dentro del segmento de pila, y apunta al último elemento introducido.

BP (índice de base)

Se usa para almacenar desplazamiento en los distintos segmentos. Por defecto es el segmento de la pila.

OF (Desbordamiento)

Indicador de error en operaciones con signo.

DF (Dirección)

Si es 1, el sentido de recorrido de la cadena es de izquierda a derecha.

IF (Interrupción)

Cuando vale 1 permite reconocer interrupciones.

TF (Modo traza)

Indica ejecución paso a paso. Se usa en depuración.

SF (Signo)

Vale 1 cuando el resultado es negativo en operaciones con signo.

ZF (Cero)

Vale 1 cuando el resultado es cero.

AF (Acarreo auxiliar)

Vale 1 cuando hay acarreo en el bit 3.

PF (Paridad)

Vale 1 si el número de bits a 1 es par.

CF (Acarreo)

Vale 1 si hay acarreo en suma o acarreo negativo en resta.

Modos de Direccionamiento

Los modos de direccionamiento indican la manera de obtener los operandos:

Registro

Ambos operandos son registros.

MOV AX,BX

Inmediato

El operando origen es una constante.

MOV AX,500

Directo

El operando es una dirección de memoria.

MOV BX,[1000]

MOV AX,TABLA

Indirecto mediante registro

El operando está en memoria en una posición contenida en un registro.

MOV AX,[BX]

Indirecto por registro base

El operando está en memoria en una posición apuntada por un registro más un desplazamiento.

MOV AX,[BP]+2

Indexado

La dirección se obtiene sumando un desplazamiento más un índice.

MOV AX,TABLA[DI]

Fig 4. Ilustración de los diferentes modos de direccionamiento

Indexado respecto a una base

La dirección se obtiene sumando un registro base, un índice y opcionalmente un desplazamiento.

MOV AX,TABLA[BX][DI]

Juego de Instrucciones

Las instrucciones del 8086/88 se pueden dividir en varios grupos:

Copian datos de un sitio a otro: MOV, XCHG, XLAT, LEA, LDS, LES, LAHF, SAHF, PUSH, PUSHF, POP, POPF.

MOV

Transfiere datos del operando origen al destino.

MOV destino, origen

PUSH/POP

Guardan y recuperan valores en la pila.

PUSH AX / POP DS

XCHG

Intercambia los valores entre dos operandos.

XCHG AX, BX

Realizan operaciones aritméticas: ADD, ADC, SUB, SBB, MUL, IMUL, DIV, IDIV, INC, DEC, NEG, etc.

ADD/ADC

Suma y suma con acarreo.

ADD AX, 34

SUB/SBB

Resta y resta con acarreo.

SUB AX, 34

MUL/IMUL

Multiplicación sin signo y con signo.

MUL BX

DIV/IDIV

División sin signo y con signo.

DIV BX

INC/DEC

Incremento y decremento.

INC AX / DEC BX

Realizan operaciones lógicas: AND, OR, XOR, NOT, TEST, CMP.

AND/OR/XOR

Operaciones lógicas bit a bit.

AND AX, BX

NOT

Negado lógico de los bits.

NOT AX

TEST

Operación AND pero sin modificar operandos.

TEST AX, BX

CMP

Compara (resta) sin modificar operandos.

CMP AX, BX

Controlan el flujo del programa: JMP, Jcc, CALL, RET, LOOP, etc.

JMP

Salto incondicional.

JMP etiqueta

Jcc

Saltos condicionales (JE, JNE, JA, JB, etc.)

JE etiqueta

CALL/RET

Llamada y retorno de procedimiento.

CALL proc / RET

LOOP

Bucle usando CX como contador.

LOOP etiqueta

Manipulan cadenas de caracteres: MOVS, CMPS, SCAS, LODS, STOS, etc.

MOVS

Copia un byte/word entre dos posiciones de memoria.

REP MOVS destino, fuente

CMPS

Compara dos cadenas en memoria.

REPE CMPS destino, fuente

SCAS

Busca un valor en una cadena.

REPNE SCAS destino

LODS/STOS

Carga/almacena en AL/AX de/a memoria.

LODS fuente / STOS destino

Etiquetas, Comentarios y Directivas

Etiquetas

Asignan un nombre a una instrucción, permitiendo hacer referencia a ellas. Máximo 31 caracteres, terminan con ":".

INICIO: MOV AX, 0

Comentarios

Describen las sentencias para facilitar comprensión. Comienzan con ";".

MOV CX, 10 ; inicializa contador

Directivas

Comandos para el ensamblador (no el procesador). Se usan para preparar segmentos, definir símbolos, reservar memoria, etc.

Directivas comunes

.MODEL

Define el modelo de memoria.

.MODEL SMALL

.STACK

Fija tamaño del segmento de pila.

.STACK 100h

.DATA/.CODE

Abren segmentos de datos y código.

.DATA / .CODE

PROC/ENDP

Definen un procedimiento.

INICIO PROC / INICIO ENDP

EQU

Asigna nombre a constantes o expresiones.

MAX EQU 100

DB/DW/DD

Reservan espacio para variables.

VALOR DB 10 / ARRAY DW 100 DUP(?)

Ejemplo de estructura básica

.MODEL SMALL
.STACK 100h
.DATA
    mensaje DB 'Hola Mundo
.CODE
MAIN PROC
    MOV AX, @DATA    ; inicializar segmento de datos
    MOV DS, AX
    
    ; código del programa
    
    MOV AH, 4Ch      ; terminar programa
    INT 21h
MAIN ENDP
END MAIN
Resumen de Implementación

Pasos para crear un programa en ensamblador

  1. Definir la estructura del programa (modelo, segmentos)
  2. Declarar variables y constantes
  3. Escribir el código del programa
  4. Ensamblar el código (generar archivo .OBJ)
  5. Enlazar el archivo objeto (generar ejecutable .EXE o .COM)

Consejos prácticos

  • Documentar el código con comentarios
  • Usar etiquetas descriptivas
  • Organizar el código en procedimientos
  • Usar instrucciones de desplazamiento (SHL/SHR) para multiplicaciones/divisiones por potencias de 2
  • Entender bien el registro de estado para saltos condicionales
  • Cuidado con el tamaño de los operandos (byte/word)
