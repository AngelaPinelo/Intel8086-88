Introducción

Guía Completa del Procesador 8086/88

Este documento proporciona una visión general completa del procesador Intel 8086/88, incluyendo su arquitectura, registros, modos de direccionamiento e instrucciones. Ideal para estudiantes y profesionales interesados en la arquitectura de computadoras y programación en lenguaje ensamblador.

Bus de datos: 16 bits (8086) / 8 bits (8088)
Bus de direcciones: 20 bits (1 MB de memoria)
Año de lanzamiento: 1978
Fabricante: Intel Corporation

El lenguaje ensamblador es un conjunto de palabras que indican al ordenador lo que tiene que hacer. A diferencia de otros lenguajes, cada instrucción en ensamblador tiene una correspondencia exacta con una operación en el procesador.

Tamaño de los datos

  • 4 bits: nibble
  • 8 bits: byte
  • 16 bits: word
  • 32 bits: dword

Almacenamiento de datos

El 8086/88 usa el formato "little endian": el byte menos significativo (LSB) del dato es guardado en la parte baja de la memoria.

0x1122
0x22
0x11

Dirección más baja → Dirección más alta

Segmentación

El 8086/88 tiene un bus de datos de 16 bits y uno de direcciones de 20 bits (accede a 1 MB). Para aprovechar la capacidad de direccionamiento, cada dirección se especifica como un segmento y un desplazamiento.

Dirección Efectiva = Segmento (desplazado 4 bits) + Desplazamiento

Registros Internos

El 8086/88 dispone de 4 registros de datos, 4 registros de segmento, 5 registros de índice y 1 registro de estado.

Los registros de datos son de 16 bits, aunque están divididos, lo que permite su acceso en 8 bits.

AX (acumulador)

AH
AL

Se usa para almacenar el resultado de las operaciones. Es el único registro con el que se puede hacer divisiones y multiplicaciones.

BX (registro base)

BX
BH
BL

Almacena la dirección base para los accesos a memoria.

CX (contador)

CX
CH
CL

Actúa como contador en los bucles de repetición. CL almacena el desplazamiento en las operaciones de desplazamiento y rotación.

DX (datos)

DX
DH
DL

Es usado para almacenar los datos de las operaciones.

Fig 2. Estructura interna de los registros del procesador 8086

Los registros de segmento son de 16 bits y contienen el valor de segmento.

CS (segmento de código)

Contiene el valor de segmento donde se encuentra el código. Actúa junto con IP para obtener la dirección de la próxima instrucción.

DS (segmento de datos)

Contiene el segmento donde están los datos.

ES (segmento extra de datos)

Es usado para acceder a otro segmento que contiene más datos.

SS (segmento de pila)

Contiene el valor del segmento donde está la pila. Se usa conjuntamente con SP para obtener la dirección del último valor almacenado.

Estos registros son usados como índices por algunas instrucciones. También pueden ser usados como operandos (excepto IP).

IP (índice de programa)

Almacena el desplazamiento dentro del segmento de código. Junto al registro CS apunta a la próxima instrucción.

SI (índice de origen)

Almacena el desplazamiento del operando de origen en memoria.

DI (índice de destino)

Almacena el desplazamiento del operando de destino en memoria.

SP (índice de pila)

Almacena el desplazamiento dentro del segmento de pila, y apunta al último elemento introducido.

BP (índice de base)

Se usa para almacenar desplazamiento en los distintos segmentos. Por defecto es el segmento de la pila.

OF (Desbordamiento)

Indicador de error en operaciones con signo.

DF (Dirección)

Si es 1, el sentido de recorrido de la cadena es de izquierda a derecha.

IF (Interrupción)

Cuando vale 1 permite reconocer interrupciones.

TF (Modo traza)

Indica ejecución paso a paso. Se usa en depuración.

SF (Signo)

Vale 1 cuando el resultado es negativo en operaciones con signo.

ZF (Cero)

Vale 1 cuando el resultado es cero.

AF (Acarreo auxiliar)

Vale 1 cuando hay acarreo en el bit 3.

PF (Paridad)

Vale 1 si el número de bits a 1 es par.

CF (Acarreo)

Vale 1 si hay acarreo en suma o acarreo negativo en resta.

Modos de Direccionamiento

Los modos de direccionamiento indican la manera de obtener los operandos:

Registro

Ambos operandos son registros.

MOV AX,BX

Inmediato

El operando origen es una constante.

MOV AX,500

Directo

El operando es una dirección de memoria.

MOV BX,[1000]

MOV AX,TABLA

Indirecto mediante registro

El operando está en memoria en una posición contenida en un registro.

MOV AX,[BX]

Indirecto por registro base

El operando está en memoria en una posición apuntada por un registro más un desplazamiento.

MOV AX,[BP]+2

Indexado

La dirección se obtiene sumando un desplazamiento más un índice.

MOV AX,TABLA[DI]

Fig 4. Ilustración de los diferentes modos de direccionamiento

Indexado respecto a una base

La dirección se obtiene sumando un registro base, un índice y opcionalmente un desplazamiento.

MOV AX,TABLA[BX][DI]

Juego de Instrucciones

Las instrucciones del 8086/88 se pueden dividir en varios grupos:

Copian datos de un sitio a otro: MOV, XCHG, XLAT, LEA, LDS, LES, LAHF, SAHF, PUSH, PUSHF, POP, POPF.

MOV

Transfiere datos del operando origen al destino.

MOV destino, origen

PUSH/POP

Guardan y recuperan valores en la pila.

PUSH AX / POP DS

XCHG

Intercambia los valores entre dos operandos.

XCHG AX, BX

Realizan operaciones aritméticas: ADD, ADC, SUB, SBB, MUL, IMUL, DIV, IDIV, INC, DEC, NEG, etc.

ADD/ADC

Suma y suma con acarreo.

ADD AX, 34

SUB/SBB

Resta y resta con acarreo.

SUB AX, 34

MUL/IMUL

Multiplicación sin signo y con signo.

MUL BX

DIV/IDIV

División sin signo y con signo.

DIV BX

INC/DEC

Incremento y decremento.

INC AX / DEC BX

Realizan operaciones lógicas: AND, OR, XOR, NOT, TEST, CMP.

AND/OR/XOR

Operaciones lógicas bit a bit.

AND AX, BX

NOT

Negado lógico de los bits.

NOT AX

TEST

Operación AND pero sin modificar operandos.

TEST AX, BX

CMP

Compara (resta) sin modificar operandos.

CMP AX, BX

Controlan el flujo del programa: JMP, Jcc, CALL, RET, LOOP, etc.

JMP

Salto incondicional.

JMP etiqueta

Jcc

Saltos condicionales (JE, JNE, JA, JB, etc.)

JE etiqueta

CALL/RET

Llamada y retorno de procedimiento.

CALL proc / RET

LOOP

Bucle usando CX como contador.

LOOP etiqueta

Manipulan cadenas de caracteres: MOVS, CMPS, SCAS, LODS, STOS, etc.

MOVS

Copia un byte/word entre dos posiciones de memoria.

REP MOVS destino, fuente

CMPS

Compara dos cadenas en memoria.

REPE CMPS destino, fuente

SCAS

Busca un valor en una cadena.

REPNE SCAS destino

LODS/STOS

Carga/almacena en AL/AX de/a memoria.

LODS fuente / STOS destino

Etiquetas, Comentarios y Directivas

Etiquetas

Asignan un nombre a una instrucción, permitiendo hacer referencia a ellas. Máximo 31 caracteres, terminan con ":".

INICIO: MOV AX, 0

Comentarios

Describen las sentencias para facilitar comprensión. Comienzan con ";".

MOV CX, 10 ; inicializa contador

Directivas

Comandos para el ensamblador (no el procesador). Se usan para preparar segmentos, definir símbolos, reservar memoria, etc.

Directivas comunes

.MODEL

Define el modelo de memoria.

.MODEL SMALL

.STACK

Fija tamaño del segmento de pila.

.STACK 100h

.DATA/.CODE

Abren segmentos de datos y código.

.DATA / .CODE

PROC/ENDP

Definen un procedimiento.

INICIO PROC / INICIO ENDP

EQU

Asigna nombre a constantes o expresiones.

MAX EQU 100

DB/DW/DD

Reservan espacio para variables.

VALOR DB 10 / ARRAY DW 100 DUP(?)

Ejemplo de estructura básica

.MODEL SMALL
.STACK 100h
.DATA
    mensaje DB 'Hola Mundo
.CODE
MAIN PROC
    MOV AX, @DATA    ; inicializar segmento de datos
    MOV DS, AX
    
    ; código del programa
    
    MOV AH, 4Ch      ; terminar programa
    INT 21h
MAIN ENDP
END MAIN
Resumen de Implementación

Pasos para crear un programa en ensamblador

  1. Definir la estructura del programa (modelo, segmentos)
  2. Declarar variables y constantes
  3. Escribir el código del programa
  4. Ensamblar el código (generar archivo .OBJ)
  5. Enlazar el archivo objeto (generar ejecutable .EXE o .COM)

Consejos prácticos

  • Documentar el código con comentarios
  • Usar etiquetas descriptivas
  • Organizar el código en procedimientos
  • Usar instrucciones de desplazamiento (SHL/SHR) para multiplicaciones/divisiones por potencias de 2
  • Entender bien el registro de estado para saltos condicionales
  • Cuidado con el tamaño de los operandos (byte/word)
